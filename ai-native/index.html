Intent

Engine

Usage

아이디어 마인드맵

창의력은 빈 화면에서 바로 나오지 않는다는 문제의식에서 출발했습니다. 사용자가 주제를 입력하면 AI가 연상어를 관련도와 의외성 기준으로 추천하고, 선택한 단어를 다시 가지처럼 확장해 아이디어를 구체화하는 발산형 도구입니다.

Intent
기획 초반의 막막함을 줄이고, 단어 기반 발산을 통해 콘텐츠·서비스 아이디어의 후보군을 빠르게 넓힙니다.
Engine
키워드를 의미 벡터처럼 다루며 관련성 높은 단어와 예상 밖의 연결어를 함께 제시해 사고의 폭을 확장합니다.
Usage
주제를 넣고 추천 단어를 누르면 하위 가지가 이어집니다. 사용자는 마음에 드는 가지를 따라가며 아이디어의 방향을 잡습니다.

HOW I BUILD

AI Native하게 만든다는 것은 프롬프트를 잘 쓰는 것보다 구조를 잘 나누는 것입니다

제가 만든 도구들은 완성도가 모두 같지는 않습니다. 대신 공통적으로 “문제를 작은 판단 단위로 쪼개고, AI가 맡을 영역과 사람이 확인할 영역을 분리한 뒤, 링크로 테스트 가능한 형태까지 만든다”는 흐름을 따릅니다.

01
병목 정의

반복 문의, 아이디어 막힘, 투자 판단, 위치 조건, 문서 작성처럼 실제로 막히는 순간을 먼저 잡습니다.

02
입출력 구조화

AI에게 던질 입력과 사용자에게 돌려줄 출력을 날짜, 조건, 키워드, 상태값으로 나눕니다.

03
AI 역할 배치

상담, 추천, 코칭, 요약, 초안화 중 AI가 잘하는 판단만 맡기고 예외는 사람에게 남깁니다.

04
프로토타입 배포

결과물을 설명서가 아니라 직접 눌러볼 수 있는 웹 링크로 만들어 검증합니다.

05
회고와 중단 판단

성공한 흐름은 고도화하고, 데이터·검증 한계가 큰 실험은 실패 이유를 남기고 멈춥니다.

EXECUTION VALUE

프로토타입으로 남긴 실행 방식

각 프로젝트는 생활 속 문제를 AI 판단 단위로 쪼개고, 필요한 도구를 조합해 실제로 돌아가는 화면과 링크로 만든 결과입니다.

POINT 01
말보다 링크가 먼저입니다

구상 단계에서 멈추지 않고 Netlify, Vercel, Cloudflare로 배포해 실제 화면과 플로우를 검증합니다.

POINT 02
AI의 역할을 제품 안에 배치합니다

LLM을 글쓰기 도구로만 쓰지 않고 상담, 추천, 코칭, 리포팅, 문서 초안화의 기능 단위로 설계합니다.

POINT 03
실패도 설계 자산으로 남깁니다

Business Plan Studio처럼 실패한 실험도 데이터 소스와 검증 한계를 기록해 다음 자동화의 기준으로 삼습니다.

AI를 잘 쓴다는 것은 일상의 문제를 작동하는 구조로 바꾸는 일이라고 생각합니다

저는 AI를 신기한 도구로 소비하기보다, 반복되는 문제를 줄이고 판단을 보조하며 실행 속도를 높이는 개인 운영체계로 다뤄왔습니다. 이 포트폴리오는 그 방식이 실제 링크와 화면으로 남은 기록입니다.
